Ssh… It’s secret…

May 10, 2022

Tags

  • SSH

Made SSH keys for a webserver rental. Been familiar with the concept but mostly used logins since machines have been local up to this point.

Mobile Putty App does not support custom keys :( Don’t like the idea of several machines being vulnerable should one system with keys be compromised. Then again, when ur pwned. Ur pwned.

Currently have a mixture of keys/logins setup.

https://www.digitalocean.com/community/tutorials/how-to-set-up-ssh-keys-on-ubuntu-20-04

On Windows

  • PuttyGen
  • Generate Keys
  • Copy Public Key
  • Create ssh file on target [ubuntu] system
    • mkdir -p ~/.ssh create .ssh file if not exist
  • Put public key contents into authorized_keys file
    • echo public_key_string_contents_here >> ~/.ssh/authorized_keys
  • Change permissions chmod -R go= ~/.ssh chown -R sammy:sammy ~/.ssh

1/15/23

Since have realized that vultr.com tutorials seem to have a more seemless directions and better success rate than digitalocean.com. This one included:

https://www.vultr.com/docs/how-to-use-ssh-with-vultr-servers/

  • ssh-keygen -t rsa
  • Follow prompts
  • cd /home/$USER
  • mkdir .ssh
  • cd .ssh
  • touch authorized_keys
  • cat /home/$USER/public-key.pub > authorized_keys
  • cd /etc/ssh
  • make sure sshd_config is setup correctly
  • systemctl restart sshd
